redis学习笔记

数据库
非关系型数据库
发布日期:   2022-04-23

1、安装

1、将redis-5.0.2.tar.gz复制粘贴到Linux中的opt目录中，解压生成一个目录，然后命令进入这个目录，使用make编译，完成后，执行make install命令，最后使用cp redis.conf /etc/redis.conf 将redis.conf复制到/etc/目录下面。

2、启动

使用redis-server /etc/redis.conf启动服务端，使用命令ps-ef | grep redis看redis服务启动没，使用redis-cli启动客户端，使用vi /etc/redis.conf修改/etc/redis.conf下面的daemonize no改成yes（位置大概是在general下面的第三四行的样子，改这个是让redis可以后台启动）。客户端启动后输入ping会输出pong

3、redis关机

单实例关闭：redis-cli shutdown

也可以进入终端后在关闭

多实例关闭，指定端口号关闭：redis-cli -p 6379 shutdown

4、常用命令：

select 数据库名（16个数据库，默认是0） 选择数据库
dbsize 查看当前数据库key的数量
flushdb 清空当前库
flushall 清空所有库

5、redis和memcache的区别：

redis与memcache****三点不同：支持多数据类型，支持持久化，使用的是单线程+多路Io复用

memcache使用的是串行+多线程+锁

6、常用五大数据类型

6.1.Redis 键(key)

key查看当前库所有key(匹配：keys1)

exists key判断某个key是否存在

type key查看你的key是什么类型

del key 删除指定的key数据

unlink key 根据value选择非阻塞删除

仅将keys从keyspace元数据中删除，真正的删除会在后续异步操作

expire key 10 10秒钟：为给定的key设置过期时间

ttl key 查看还有多少秒过期，-1表示永不过期，-2表示已经过期

select命令切换数据库

dbsize查看当前数据库中所有的key的数量

flushdb清空当前库

flushall通杀全部库

6.2.key的操作

先登录客户端，使用set k1 lucy这样的命令设置值

6.3.Redis 字符串String

String是Redis最基本的类型，是二进制安全的

6.3.1. 常用命令

set 添加键值对

get

append 将给定的追加到原值的末尾

6.4 redis数据类型-list

lpush/rpush key 从左边插入，从右边插入。

lpop/rpop key 从左边/右边吐出一个值。值在键在，值光键亡

rpoplpush k1 k2 从k1右边弹出一个值加到k2的左边

lrange k1 0 -1 按照索引获得元素（从左到右）0表示左边第一个，-1表示右边第一个

lindex k1 0 获取指定索引处的值

llen 获取列表长度

lset key index value 将列表key下标为index的值替换成value

linsert key before/after value newvalue 在value的前面/后面加上newvalue的值

lrem k1 2 “newvalue” 从左到右删除n个newvalue

list数据结构

6.5 redis数据类型-set

sadd key value1 value2 value3 将一个或多个member元素加入到集合key中 ，已经存在的member元素将被忽略

smembers key取出该集合的所有值

sismember key value 判断key是否含有该value值，有1，没有就是0

scart key 返回该集合的元素个数

srem key value1 value2 删除集合中某个元素

spop key 随机从该集合中吐出一个值

srandmember key n 随机从集合中取出n个值，不会从集合中删除

smove value 把集合中一个值从一个集合中移动到另一个集合

sinter k1 k2 返回两个集合的交集

sunion k1 k2 返回两个集合的并集

sdiff k1 k2 返回两个集合的差集元素（k1中的，不包括k2中的)

set数据结构

set数据结构是dict字典，字典是用哈希表实现的

6.6 redis数据类型hash

6.6.1 常用命令

hset 给集合中的键赋值

hget 从集合取出value

hmset 批量设置hash的值

hexists查看哈希表key中，给定域field是否存在

hkeys列出该hash集合中的所有field

hvals 列出该hash集合的所有value

hincrby为哈希表key中的域field的值加上增量 1 -1

hsextnx 将哈希表中的域field的值设置为value，当且仅当域field不存在.

6.6.2 hash的数据结构

Hash类型对应的数据结构是两种：ziplist(压缩列表),hashtable(哈希表)。当field-value长度较短且个数较少时，使用ziplist，否则使用hashtable

6.7 redis数据类型-zset

zadd …

将一个或多个member元素及其score值加入到有序集合key当中

zrange key start stop 【withscores]，返回有序集key中，下标在start和stop之间的元素，带withscores，可以让分数和值返回到结果集

zincrby 为元素的score加上增量

zrem 删除该集合下，指定值的元素

zcount统计该集合，分数区间内的元素个数

zrank 返回该值在集合中的排名，从0开始

7 redis事务：

7.1 事务介绍

multi是开启事务，exec是执行，discard相当于回滚。

结论：如果组队阶段有错误，那么exec执行会导致所有操作都不成功

如果组队阶段没错误，exec执行阶段报错，报错的操作不成功，其他的成功。

第一版的controller层

package com.zdh.test.controller;

import com.zdh.test.service.TestService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.autoconfigure.data.redis.RedisProperties;
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.ResponseBody;
import org.springframework.web.servlet.ModelAndView;
import redis.clients.jedis.Jedis;
import redis.clients.jedis.JedisPool;
import redis.clients.jedis.Transaction;
import java.util.List;
import java.util.UUID;

@Controller
public class TestController {
    @RequestMapping("/hh")
    @ResponseBody
    public boolean secKill(Integer ppid){
        if (ppid == null){
            System.out.println("商品空");
        }
        String uid = UUID.randomUUID().toString();
        String kcKey = "sk:"+ppid+":qt";
        String userKey = "sk:"+ppid+":user";
        Jedis jedis = new Jedis("192.168.66.128",6379);
        String s = jedis.get(kcKey);
        if (s == null){
            System.out.println("没开始");
            jedis.close();
            return false;
        }
        if (jedis.sismember(userKey,uid)){
            System.out.println("抢过了");
            jedis.close();
            return false;
        }
        if (Integer.valueOf(s) < 1){
            System.out.println("秒杀结束");
            jedis.close();
            return false;
        }
   
        jedis.decr(kcKey);
        jedis.sadd(userKey,uid);
        System.out.println("秒杀成功");
        jedis.close();
        return true;
    }
}

出现的问题？

出现了超卖和超时问题，原因：超时问题是因为一个redis无法处理那么多的请求，超卖问题，是因为没有事务的控制。

第二版：利用事务解决超卖问题，利用redis连接池解决超时问题。

第二版的controller

package com.zdh.test.controller;

import com.zdh.test.service.TestService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.autoconfigure.data.redis.RedisProperties;
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.ResponseBody;
import org.springframework.web.servlet.ModelAndView;
import redis.clients.jedis.Jedis;
import redis.clients.jedis.JedisPool;
import redis.clients.jedis.Transaction;

import java.util.List;
import java.util.UUID;

@Controller
public class TestController {
    @Autowired
    private JedisPool jedisPool;

    @RequestMapping("/hh")
    @ResponseBody
    public boolean secKill(Integer ppid){
        if (ppid == null){
            System.out.println("商品空");
        }
        String uid = UUID.randomUUID().toString();
        String kcKey = "sk:"+ppid+":qt";
        String userKey = "sk:"+ppid+":user";
        //Jedis jedis = new Jedis("192.168.66.128",6379);
        Jedis jedis = jedisPool.getResource();
        //监视库存
        jedis.watch(kcKey);

        String s = jedis.get(kcKey);
        if (s == null){
            System.out.println("没开始");
            jedis.close();
            return false;
        }
        if (jedis.sismember(userKey,uid)){
            System.out.println("抢过了");
            jedis.close();
            return false;
        }
        if (Integer.valueOf(s) < 1){
            System.out.println("秒杀结束");
            jedis.close();
            return false;
        }
        Transaction multi = jedis.multi();
        //组队
        multi.decr(kcKey);
        multi.sadd(userKey,uid);
        //执行
        List<Object> results = multi.exec();
        if (results == null || results.isEmpty() || results.size() <= 0){
            System.out.println("秒杀没成功");
            jedis.close();
            return false;
        }
        System.out.println("秒杀成功");
        jedis.close();
        return true;
    }
}

出现问题的原因是乐观锁，操作数据时，回去比对版本号，版本号不一致就不进行。

解决方案：使用lua脚本。
